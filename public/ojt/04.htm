서버(Server)

소리내어 읽기그만읽기
 

도메인에서 한 번 다뤘는데, 정확하게 다시 정의하자면, 서버는 클라이언트에게 네트워크를 통해 정보나 서비스를 제공하는 컴퓨터 또는 프로그램을 뜻합니다. 잘 이해되시겠죠?

  • 웹서버(Web Server)
    http 서비스를 위한 서버를 의미합니다. 클라이언트로부터 요청을 받으면 html 문서를 보내줍니다. 웹서버 프로그램을 의미하기도 하고, 웹서버 프로그램 작동이 주된 역할인 컴퓨터를 의미하기도 합니다.
    아파치(apache)나 iis 등이 대표적인 웹서버입니다. 요즘엔 nginx도 많이 씁니다. n.mp1.co.kr은 nginx를 씁니다.
  • 웹어플리케이션서버(WAS; Web Application Server)
    예를 들어, 데이터베이스에 있는 정보를 가져와 클라이언트가 원하는 형태로 문서를 만드는 프로그램이 있다고 합시다. 그 프로그램을 작동시켜 웹서버를 통해 클라이언트에게 전달하는 것이 WAS입니다.
    엄밀하게는 웹컨테이너와 웹서버를 합한 것을 WAS라 하는데, 그렇게까지 기억해둘 필요는 없으십니다.
    mp1에는 톰캣(tomcat)을 사용합니다. 규칙만 지키면 무료이고, 자바언어로 개발된 웹용 프로그램을 돌릴 수 있습니다.
  • 디비서버(Database Server)
    데이터베이스 관리 프로그램(DBMS; Database Management System)이 설치된 서버를 의미합니다.
  • 이미지서버(Image Server)
    html에 사용되는 정적 데이터들을 모아둔 웹서버입니다. 웹서버의 부하(traffic)를 줄이기 위해 따로 구성합니다. mp1은 image.mp1.co.kr로 분리되어 있습니다.
  • 정적 데이터? 동적 데이터?
    변경되지 않는 데이터를 정적 데이터라 합니다. 주로 이미지, 동영상 등입니다.
    클라이언트의 요청에 따라 변경되는 데이터를 동적 데이터라 합니다. 주로 검색 조건이나 시간에 따라 데이터베이스에서 결과를 가져와 제공되는 데이터를 의미합니다. 보통, 웹개발자는 이 동적 데이터를 다루는 사람들을 의미합니다.
    일반적으로 정적 데이터는 웹서버에서, 동적 데이터는 웹어플리케이션서버에서 처리합니다. 당연하겠죠?
  • 캐시서버(Cache Server)/CDN(Content Delivery Network)
    아무리 네트워크 환경이 좋아도 미국에 서버를 두고 서비스하는 것과 한국에 서버를 두고 하는 것은 속도의 차이가 날 수 밖에 없습니다. 그래서, 사용자 가까이에 제공할 컨텐츠를 복제해서 제공하는 서버를 만들기도 하는데, 이것을 캐시서버라 합니다. 캐시서버에서 제공할 수 없는 새로운 요청만 원래의 서버에 요청하게 구성합니다. 이러한 구성을 CDN(Content Delivery Network)라 합니다.
  • IDC(Internet Data Center)
    인터넷데이터센터(IDC)는 서버 컴퓨터를 안전하게 보관할 장소와 빠른 네트워크 회선 등을 제공하는 시설을 말합니다. 서버호텔이라고 이해하시면 됩니다. 들어가보면 컴퓨터가 가득합니다. 시끄럽고 정말 춥고 건조합니다. 전자파로 샤워하는 느낌입니다. 케이지를 만들고, 그 안에 랙을 넣고, 랙에 스위치와 서버를 차곡차곡 쌓아둡니다.
    처음앤씨는 역삼동에 있는 IDC를 이용합니다. 사무실의 서버룸은 통신설비와 내부용 서버 등이 들어 있습니다.
  • 분산 네트워크 서비스
    클라이언트 수가 늘어나면 수십대의 컴퓨터로도 감당이 안되기 시작합니다. 분산이라는 말은, 트래픽이나 데이터 등을 나눈다는 것을 의미합니다. 보통 웹서버와 웹어플리케이션서버부터 분산을 시작하고, 데이터까지 분산해야할 정도면... 부자가 되신겁니다.
  • 서버용 컴퓨터는 PC와 다른가?
    PC를 서버로 써도 됩니다. 다만, 랙에 들어갈 수 있도록 제작되고, 목적에 맞게 성능이 강화된 컴퓨터를 서버용 컴퓨터로 보통 부릅니다.
    사진은 제가 좋아하는 hp사의 580서버입니다. 분홍색(?) 하나에 하드디스크 하나를 꼿을 수 있다고 보시면 됩니다.
  • 하나의 컴퓨터에 웹서버, 웹어플리케이션서버, 디비서버를 다 올려놓고 써도 되지 않나?
    됩니다. 사용자가 많으면 그때 후회해도 됩니다.
